## 클레이튼 v1.3.0 릴리스 노트

## 공지사항 <a id="notice"></a>

- 보안 문제를 해결하기 위해 klaytn v1.3.0의 공식 go 버전은 go-1.13.3입니다. klaytn을 빌드할 때는 go-1.13.3 이상 버전을 권장합니다.

### 새로운 기능 <a id="new-features"></a>

- RPC API를 통한 Klaytn 트랜잭션 유형 서명(klay_signTransaction 및 klay_signTransactionAsFeePayer).
- TxTypeFeeDelegatedChainDataAnchoring과 TxTypeFeeDelegatedChainDataAnchoringWithRatio가 추가되었습니다. 이제 서비스 체인에서 앵커링 트랜잭션에 대한 트랜잭션 수수료를 다른 수수료 납부 계정으로 지불할 수 있습니다.
- 새로운 JSON-RPC API
  - klay.encodeAccountKey
  - klay.decodeAccountKey
  - klay.sendTransactionAsFeePayer
  - subbridge.setParentOperatorFeePayer
  - subbridge.parentOperatorFeePayer
  - subbridge.setChildOperatorFeePayer
  - subbridge.childOperatorFeePayer

### 수정 <a id='fixes'></a>

- 상태 업데이트 건너뛰기로 인한 잘못된 불량 블록 발생을 수정했습니다.
- call/estimateGas/estimateComputationCost 수행 시 발신자의 잘못된 잔액을 반환하는 문제를 수정했습니다.
- 시간 초과 전에 대기 중인 트랜잭션이 손실되는 현상을 수정했습니다.

### 개선 사항 <a id='improvements'></a>

- 더 빠른 복구를 위해 라운드 변경 타이머를 3초로 수정했습니다.
- CN 간의 컨센서스 메시지 전파 최적화
- 안정성 향상을 위해 테스트 코드 추가
- 패키지 설명 추가
- 개선된 패키징 스크립트

## 제거 <a id='removals'></a>

- 사용하지 않는 패키지 'filter'를 제거했습니다.
- 사용하지 않는 JSON-RPC API 제거
  - "miner" 네임스페이스의 API
  - klay.mining
  - klay.submitWork
  - klay.getWork
  - klay.submitHashrate
