## 릴리스 노트

### 새로운 기능
- klay_getLogs 및 klay_getFilterLogs API에 실행 기한 및 최대 항목 제한 도입 (#903)
- 해시잠김 상태를 기록하는 메트릭 도입 (#932)
- 스테이킹 정보 API(governance.getStakingInfo) 추가 (#929)
- HTTP-RPC 서버의 동시성 제한을 제한하는 옵션(rpc.concurrencylimit) 도입 (#901) (#901)

### 개선 사항
- 해시 잠금 문제 해결을 위한 COMMIT 메시지 전송 조건 추가 (#901)
- 프리페치
  - 한 블록에서 동일한 발신자의 여러 tx에 대한 프리페치 로직 수정 (#901)
  - 여러 블록에 대해 TX 프리페처 활성화 (#901)
  - 프리페처 실행을 가능한 한 빨리 이동 (#901)
  - 트랜잭션 액세스의 경쟁 조건 수정 (#901)

### 수정
- 블록 헤더의 추가 데이터에 잘못된 유효성 검사기 목록 수정 (#901)

### 기타
- API 이름 변경, setMaxSubscriptionPerConn에서 setMaxSubscriptionPerWSConn으로 변경 (#901).
- 하드포크 이름 변경, Incompatible1에서 IstanbulCompatible으로, 문서 업데이트 필요 (#901)
