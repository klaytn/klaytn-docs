## 릴리스 노트

### 새로운 기능

- 클레이튼 엔드포인트 시스템(KES) 도입 (#707, #709)
  - 공유 데이터베이스 및 캐시 지원
  - KES 페처 노드는 블록을 처리하고, KES 서비스 노드는 별도의 처리 없이 공유 데이터베이스/캐시로 API를 서비스합니다.
- 상태 시도 통계를 표시하는 debug.startCollectingTrieStats API 추가(#740)
- 컨트랙트 소트라지에서 수정된 트라이 노드를 반환하는 debug.getModifiedStorageNodesByNumber API를 추가했습니다(#742).
- 블록 처리 요약을 보여주는 로그 추가 (#747)
- unigram/bigram/trigram 트레이서 추가 및 4바이트 트레이서 오류 수정 (#752)
- 동적 배열 및 슬라이스 유형 패킹을 위해 ABIEncoderV2 지원(#753)
- 튜플 타입 및 바이트 배열의 언팩을 위한 ABI 지원(#807, #792, #786).
- 파일시스템에 주기적으로 빠른 캐시를 저장하기 위한 state.trie-cache-save-period 옵션 도입(#828).
- 컨트랙트 스토리지 트라이 노드를 워밍업하기 위한 debug.StartContractWarmUp(common.Address) API 추가 (#831).
- 상태 트라이 캐시를 미리 로드하기 위해 현재 상태에 대해 미래 블록과 미래 트랜잭션을 실행하는 상태 프리페처 도입 (#832, #860).

### 개선 사항

- 체인 데이터를 분할하여 체인 데이터 페처로 Kafka 메시지 전송(#708, #712)
- metrics.Meter를 metrics.Timer로 변경 (#817)
- 기본 도커 이미지 ID 업데이트 (#824)
- FD 한도를 프로세스의 최대값으로 증가시켰습니다(#827).

### 수정

- 트라이 노드 캐시 저장 경로 수정 (#751)

### 기타

- LevelDB에 대한 검색 압축 비활성화(#715)
- 코드 리팩토링 수행(#730, #733, #735)
- golang을 1.15.7 버전으로 업데이트(#875, #871)
