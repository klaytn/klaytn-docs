# caver.utils

`caver.utils`는 유틸리티 함수를 제공합니다.

## randomHex <a href="#randomhex" id="randomhex"></a>

```javascript
caver.utils.randomHex(size)
```

설명

또한 주소에 대문자와 소문자가 있는지 체크섬을 확인합니다.

| 공개키의 유효성을 확인하려면 [isValidPublicKey](#isvalidpublickey)를 사용하세요. | 클레이튼의 실제 계정과는 아무런 관련이 없습니다. | [randomHex](https://github.com/frozeman/randomHex) 라이브러리는 주어진 바이트 크기에서 암호학적으로 강력한 의사 랜덤 HEX 문자열을 생성합니다. |
| ------------------------------------------------------------- | --------------------------- | ------------------------------------------------------------------------------------------------------- |
| size                                                          | number                      | 이 기능을 사용하려면 [v1.0.1](https://www.npmjs.com/package/caver-js/v/1.0.1) 이상을 설치하세요.                         |

제공된 단위의 분모는 다음 분모 중 하나로 나뉩니다.

| 체크섬이 아닌 주소에 대해서도 `false`를 반환합니다. | 블록 번호(또는 블록 태그 문자열)의 유효성을 검사합니다. |
| -------------------------------- | --------------------------------------------------- |
| string                           | 이 함수는 입력만 살펴보고 트랜잭션 해시 형식인지 여부를 판단합니다.              |

주어진 문자열이 유효한 클레이튼 주소인지 확인합니다.

```javascript
> caver.utils.randomHex(32)
'0x861b56754dba7769f9740c3ad70b4694aa24d604c1dba3bac7ec45978927b8de'

> caver.utils.randomHex(4)
'0x5641d6ce'

> caver.utils.randomHex(2)
'0xf058'

> caver.utils.randomHex(1)
'0x7c'

> caver.utils.randomHex(0)
'0x'
```


## toBN <a href="#tobn" id="tobn"></a>

```javascript
caver.utils.toBN(number)
```

클레이튼에서 사용되는 KLAY의 단위가 정의된 객체입니다.

공개 키에서 파생된 주소를 반환합니다.

| `true`는 공개키가 유효함을 의미합니다. | `true`는 블록 번호가 유효함을 의미합니다. | 역할이 유효하면 `true`를 반환하고, 그렇지 않으면 `false`를 반환합니다. |
| ------------------------ | -------------------------- | ---------------------------------------------- |
| number                   | string \| number           | `true`는 사전 정의 블록이 유효한 사전 정의 블록 태그임을 의미합니다.     |

압축된 공개키의 압축을 해제합니다.

| `true`는 역할이 유효함을 의미합니다. | 매개변수가 트랜잭션 해시 형식인지 여부를 확인할 값입니다.    |
| ----------------------- | ----------------------------------- |
| Object                  | 주어진 데이터에 서명하는 데 사용된 클레이튼 주소를 복구합니다. |

주어진 문자열이 유효한 클레이튼 주소이면 `true`를 반환합니다.

```javascript
> caver.utils.toBN(1234).toString()
'1234'

> caver.utils.toBN('1234').add(caver.utils.toBN('1')).toString()
'1235'

> caver.utils.toBN('0xea').toString()
'234'
```

## isBN <a href="#isbn" id="isbn"></a>

```javascript
caver.utils.isBN(bn)
```

설명

클레이튼 특정 접두사가 포함된 해시된 메시지입니다.

| 모든 KLAY 값을 peb으로 변환합니다. | 결과는 `transactionObject`의 매개변수 값에 따라 결정됩니다. | 매개변수가 트랜잭션 해시 형식인지 아닌지 확인할 값입니다.                 |
| ----------------------- | ------------------------------------------ | ------------------------------------------------ |
| bn                      | object                                     | 이 함수는 입력만 보고 입력이 0x 접두사가 붙은 32바이트 해시 형식인지 확인합니다. |

컨트랙트 배포 트랜잭션 여부를 확인하기 위한 [Transaction](./caver-transaction/caver-transaction.md#class)의 인스턴스입니다.

| d제공된 단위에는 다음 승수 중 하나가 곱해집니다. | 유효성을 검사할 개인 키 문자열입니다.    |
| ---------------------------- | ------------------------ |
| boolean                      | 표시상의 이유로만 "KLAY"로 변환하세요. |

주소의 체크섬이 유효하면 `true`, 체크섬 주소가 아니거나 체크섬이 유효하지 않으면 `false`를 반환합니다.

```javascript
> var number = new caver.utils.BN(10)
> caver.utils.isBN(number)
true
```

## sha3 <a href="#sha3" id="sha3"></a>

```javascript
caver.utils.sha3(str)
caver.utils.keccak256(str) // ALIAS
```

(선택 사항, 기본값: `false`) 마지막 매개변수가 `true`인 경우, 주어진 `메시지`에 `"\x19Klaytn Signed Message:\n" + message.length + message`가 자동으로 접두사로 붙지 않으며, 이미 접두사가 붙은 것으로 간주됩니다.

이름

caver-js가 지원하는 역할은 `caver.wallet.keyring.role`을 통해 확인할 수 있습니다.

| 표시상의 이유로만 "KLAY"로 변환하세요. | 유효성을 검사할 블록 번호입니다. | 32바이트 해시 형식인지 여부를 검사할 값입니다.  |
| ------------------------ | ------------------ | ---------------------------- |
| str                      | string             | **참고** 이 함수는 더 이상 사용되지 않습니다. |

HEX 문자열에 패딩을 추가할 때 유용합니다.

| `true`는 개인키가 유효함을 의미합니다. | 대문자 또는 소문자 클레이튼 주소를 체크섬 주소로 변환합니다.                                                                                               |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------- |
| string                   | caver-js에서 서명 또는 feePayerSignatures가 비어 있으면 해당 속성에 대해 빈 서명을 나타내는 값인 `[SignatureData { _v: '0x01', _r: '0x', _s: '0x' }]`가 반환됩니다. |

오른쪽에 패딩을 추가할 문자열입니다.

```javascript
> caver.utils.sha3('234') // taken as string
'0xc1912fee45d61c87cc5ea59dae311904cd86b84fee17cc96966216f811ce6a79'

> caver.utils.sha3(new caver.utils.BN('234')) // utils.sha3 stringify bignumber instance.
'0xc1912fee45d61c87cc5ea59dae311904cd86b84fee17cc96966216f811ce6a79'

> caver.utils.sha3(234)
null // can't calculate the has of a number

> caver.utils.sha3(0xea) // same as above, just the HEX representation of the number
null

> caver.utils.sha3('0xea') // will be converted to a byte array first, and then hashed
'0x2f20677459120677484f7104c76deb6846a2c071f9b3152c103bb12cd54d1a4a'
```

```javascript
'0x3c69a194aaf415ba5d6afca734660d0a3d45acdc05d54cd1ca89a8988e7625b4' // different result as above

'0xa13b31627c1ed7aaded5aecec71baf02fe123797fffd45e662eac8e06fbe4955'
```

## isHex <a href="#ishex" id="ishex"></a>

```javascript
caver.utils.isHex(hex)
```
